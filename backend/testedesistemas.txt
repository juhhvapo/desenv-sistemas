ATIVIDADE 

Cenário: Aiq Fome 

- Liste 10 coisas que poderiam dar errado no sistema

- Problemas de localização: O aplicativo não passa a localização correta para o entregador 
- Problemas de pedidos: A loja não está fazendo o produto e o aplicativo da a opção de pedir
- Pagamento não Processado: O cliente paga mas o sistema não notifica o pagamento 
- Falha no Sistema: Cartão de crédito ou dados pessoais do cliente são vazados
- Falha de Comunicação: O aplicativo falha na comunicação com o entregador e com a loja fazendo o pedido atrasar
- Falha no servidor: o aplicativo fica fora do ar e impede os usuários de fazer pedidos 
- Notificação falhando: O usuário não recebe confirmação do pedido ou aviso de entrega
- Erros de Integração com restaurantes: O pedido chega incompleto ou com informações erradas
- Pedidos duplicados: O usuário clica uma vez, mas o pedido é enviado duas vezes 
- Erro no cálculo do valor total: O aplicativo soma errado taxas, descontos ou frete

TÉCNICAS DE TESTES - AS "CAIXAS"

- Caixa Preta - O "tester" não conhece o código. Testa somente a funcionalidade

- Caixa Branca - O "tester" tem acesso total ao código para realizar os testes e sugerir correções

- Caixa Cinza - O "tester" tem um breve Conhecimento do código e testa a funcionalidade

O QUE É UM PLANO DE TESTES?

- É o "mapa" que guia a equipe de testes 
- Define "o que", "como", "quando" e "quem" irá realizar os testes
- Contém todos os testes que serão realizados naquele momento

O QUE É UM CASO DE TESTE?

- É a menor unidade dos testes
- Realizar um teste em particular, de forma específica

ANATOMIA DE UM CASO DE TESTE

- Todo caso de teste eficaz deve ter:
- ID único              - Passos para execução        - Resultado obtido
- Título descritivo     - Dados de testes
- Pré-condições         - Resultado esperado

EXEMPLO:

ID: CT(caso de teste)-gmail- 001
TÍTULO: Login com usuário e senha válidos
PRÉ-CONDIÇÕES: Estar na página inicial de login do gmail
               Possuir uma conta de usuário e senha válidos

Passos: Digitar o e-mail
        Digitar a senha
        Clicar em "entrar"

Dados do teste: e-mail: usuário@gmail.com
                senha: teste123

Resultado esperado: O usuário deve ser autenticado e direcionado para a tela inicial do gmail

Resultado obtido: O teste funcionou como o esperado

O QUE É UM BUG REPORT?

- É um documento formal que comunica um erro encontrado

- Permite que o desenvolvedor entenda e reproduza o erro

MANTRA/DITADO DE QA: "Um bug que não pode ser reproduzido não pode ser corrigido"

PORQUE ESSE BUG REPORT É INUTIL?

Títilo: o botão quebrou
Descrição: Fui criar uma tarefa e o botão salvar não funciona

O QUE FALTA:

Qual botão? em que tela? o que significa "não funciona"?
Qual era o resultado esperado?
